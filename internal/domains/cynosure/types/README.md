# Types

Компоненты — это базовые кубики лего, из которых собираются все другие части. Их смысл — обозначить не только физически, за что отвечает какая-то сущность, но и логически, по названию типа, показать какой смысл у этих данных.

Компоненты это самые главные примитивы в рамках доменной модели. Любой параметр, любой аргумент, вообще всё, что имеет смысл кроме какого-то значения самого по себе — должно быть компонентом, при этом чётко привязанным к смыслу.

Например, для создания IAM системы нам требуется сделать пользователя, а у пользователя есть параметры “имя“ и “фамилия”. Хоть и с точки зрения фактических данных, это просто две строки, с точки зрения нас, как разработчиков, и бизнеса, как заказчиков — это достаточно абстрактная еденица, в рамках которой не важно как она хранится — набор UTF байт, base64, или же сжатый набор данных. Имя — это имя.

Типы неизменяемы по своей сути. Однажды созданный тип не должен меняться
никогда. Это только read only объект, и что бы его модифицировать — он должен
быть скопирован, и модифицирована должна быть только его копия.
